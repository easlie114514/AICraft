# AI 情绪表达窗口 — 需求设计文档

> 文档状态：需求确认中
> 创建日期：2026-06-27
> 产品定位：AICraft "平民娱乐级"路线

---

## 一、功能概述

在 AICraft 聊天界面中，为每个角色加入**像素风 AI 情绪画像**。每次 AI 回复完成后，画像自动切换为对应的表情。灵感来源：星露谷物语、逆转裁判等像素游戏中的角色对话框。

**核心体验**：一种轻量、有趣、不打扰的 UI 趣味元素。用户发消息 → AI 回复 → 画像瞬间切换表情（就像游戏 NPC 对话时立绘变化一样）。

---

## 二、设计规格

### 2.1 视觉风格

- **参考**：星露谷物语 / 逆转裁判 123 的像素角色立绘风格
- **表现**：像素画角色头像，不同表情**瞬间切换**（无过渡动画，纯游戏对话框体验）
- **技术**：6 张独立 PNG 图片，文件存在即已配置，切换时直接换 `<img src>`，通过预加载避免闪烁
- **渲染**：`image-rendering: pixelated` 保持像素边缘锐利
- **尺寸参考**：星露谷物语原版对话立绘为 **64×64px** 每帧。前端实际显示时可等比放大到 2×~3×（128×128 ~ 192×192 的 CSS 显示尺寸）

### 2.2 显示位置

```
┌──────────────────────────────────────────┐
│           聊天消息区域                     │
│                                           │
│  [AI 回复文本...]                         │
│                                           │
│  [AI 回复文本...]  ← 最后一条回复          │
│                                           │
├──────────────────────────────────────────┤
│ ┌──────┐ ┌─────────────────────────────┐ │
│ │      │ │                             │ │
│ │ 画   │ │  消息输入框                   │ │
│ │ 像   │ │                             │ │
│ │      │ │                             │ │
│ └──────┘ └─────────────────────────────┘ │
└──────────────────────────────────────────┘
```

- 画像位于输入框**左侧**，固定矩形区域（约 80×96px）
- 输入框整体向右缩进，腾出画像空间
- 画像显示的是**最后一次 AI 回复完成后的情绪**（流式输出期间的中间状态不反映到画像上）

### 2.3 情绪集合（6 种，暂定）

| 序号 | 情绪 key | 表情描述 | 触发场景 |
|------|----------|---------|---------|
| 1 | `neutral` | 平静脸，默认表情 | 会话开始、一般对话 |
| 2 | `happy` | 开心 / 微笑 | 正常完成任务、用户称赞 |
| 3 | `thinking` | 思考中（手托下巴或眯眼） | 处理复杂问题、多步推理 |
| 4 | `confused` | 疑问（歪头 / 问号） | 用户输入不明确、需要澄清 |
| 5 | `working` | 专注干活 | 执行工具调用、搜索中 |
| 6 | `concerned` | 担忧 / 歉意 | 检测到风险、无法完成任务 |

### 2.4 画像配置（角色级，手动导入）

情绪画像**按角色配置**，位于角色创建/编辑页面。

#### 2.4.1 开关

- 角色编辑页中有一个 **"情绪画像"** 开关，**默认关闭**
- 打开后展开 6 个情绪槽位的配置区域

#### 2.4.2 6 个情绪预览槽位

```
┌─────────────────────────────────────────────────┐
│  ☑ 情绪画像  [开关]                              │
│                                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │          │ │          │ │          │        │
│  │   😊     │ │   🤔     │ │   😕     │        │
│  │  happy   │ │ thinking │ │ confused │        │
│  │  (已配)   │ │  (已配)   │ │  (空)    │        │
│  └──────────┘ └──────────┘ └──────────┘        │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │          │ │          │ │          │        │
│  │   😐     │ │   ⚙️     │ │   ⚠️     │        │
│  │ neutral  │ │ working  │ │concerned │        │
│  │  (已配)   │ │  (空)    │ │  (已配)   │        │
│  └──────────┘ └──────────┘ └──────────┘        │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 6 个槽位，每个对应一种情绪
- **已配置**的槽位显示裁剪后的预览图
- **空槽位**显示虚线边框 + 情绪名称，等待用户配置
- 点击任意槽位 → 进入该情绪的**裁剪配置弹窗**

#### 2.4.3 单情绪裁剪配置弹窗

```
┌─────────────────────────────────────┐
│  配置 happy 表情                      │
│                                     │
│  [ 选择图片 ]  (从本地导入)            │
│                                     │
│  ┌─────────────────────────────┐    │
│  │                             │    │
│  │     导入的原始图片            │    │
│  │                             │    │
│  │        ┌──────┐             │    │
│  │        │ 固定  │  ← 64×64   │    │
│  │        │ 选框  │    裁剪框   │    │
│  │        └──────┘             │    │
│  │                             │    │
│  └─────────────────────────────┘    │
│                                     │
│  ┌──────┐                           │
│  │ 预览  │  裁剪后的 64×64 像素       │
│  └──────┘                           │
│                                     │
│       [取消]    [确认保存]            │
└─────────────────────────────────────┘
```

**交互流程**：
1. 用户点击槽位 → 弹出配置窗口
2. 点击"选择图片"从本地导入任意尺寸图片
3. 图片上叠加一个**固定 64×64 像素的裁剪框**
4. 用户拖拽移动裁剪框（或缩放图片）选择目标区域
5. 右侧实时预览裁剪后的 64×64 像素结果（`image-rendering: pixelated`）
6. 确认保存 → 裁剪帧写入 `roles/<role>/emotion_<key>.png`

#### 2.4.4 空槽位处理（关键）

- 如果某个情绪的槽位为空 → **该情绪不加入 LLM 上下文**
- 即 LLM system prompt 中的可选情绪列表**动态生成**，仅列出已配置帧的情绪
- 例如：用户只配了 `neutral`、`happy`、`thinking` 三帧，那么 prompt 中仅出现这三个选项
- 如果**所有槽位都为空**（用户打开了开关但还没配置任何帧），等同于关闭，LLM 不做情绪判断

#### 2.4.5 存储

- 每个角色的情绪画像保存在角色目录下，每帧一张独立 PNG：
  ```
  roles/<role_name>/
    emotion_neutral.png
    emotion_happy.png
    emotion_thinking.png
    emotion_confused.png
    emotion_working.png
    emotion_concerned.png
  ```
- 每帧规格：**64×64px**
- **文件存在即已配置**，不存在即为空槽位，无需额外 manifest
- 同时保存开关状态：`roles/<role_name>/emotion.json`
  ```json
  {
    "enabled": true
  }
  ```
- 简化设计：enabled + 文件存在性共同决定哪些情绪可用

### 2.5 全局显示开关

- 设置页面提供 **"显示 AI 情绪画像"** 全局开关，默认**开启**
- 关闭后：即使角色配置了情绪画像，聊天页也不显示画像，输入区域恢复全宽
- 与 2.4.1 的角色级开关是**两层控制**：角色级决定"配不配"，全局级决定"看不看"

## 二点五、角色存储架构变更

### 现状

角色以**单文件** `.md` 存储，角色名 = 文件名（stem）：

```
roles/
  AICraft智能小助手.md      ← 一个角色 = 一个 .md 文件
  MyRole.md
```

`RoleLoader.scan()` 扫描 `roles/*.md`，`Role.path` 指向 `.md` 文件。

### 目标

角色改为**文件夹**存储，角色名 = 文件夹名：

```
roles/
  AICraft智能小助手/         ← 一个角色 = 一个文件夹
    role.md                  ← 系统提示词（必选）
    emotion.png              ← 情绪精灵图（可选，仅开启情绪画像时存在）
    emotion.json             ← 情绪 manifest（可选，记录哪些帧已配置）
  MyRole/
    role.md
    emotion.png
    emotion.json
```

### 影响范围

| 模块 | 变更 |
|------|------|
| `src/core/role_loader.py` | `scan()` 改扫 `roles/*/role.md`；`Role.path` 指向文件夹 |
| `backend/routers/roles.py` | CRUD 操作文件名 → 文件夹名 |
| `frontend/src/pages/RolePage.tsx` | 无感知（API 接口保持不变，仍以 name 标识） |
| `backend/chat_ws.py` | `build_system_prompt()` 读取 `role.md` 路径变更 |

### 迁移策略

开发时做**自动迁移**，对用户透明：

1. `RoleLoader.scan()` 同时兼容旧格式（`roles/*.md`）和新格式（`roles/*/role.md`）
2. 发现旧格式 `.md` 文件时，自动创建同名文件夹，移入 `role.md`，删除旧文件
3. 如果新旧格式同名角色同时存在，新格式优先（旧格式视为待迁移残留，跳过）

### 3.1 数据流

```
┌─ 画像配置（角色编辑时，一次性）──────────────┐
│                                              │
│  角色编辑页 打开 "情绪画像" 开关               │
│    → 展开 6 个槽位                            │
│    → 点击某槽位 → 裁剪弹窗                     │
│    → 导入图片 → 固定框裁剪 64×64              │
│    → 保存到 roles/<role>/emotion_<key>.png       │
│    → 更新 roles/<role>/emotion.json 开关状态     │
└──────────────────────────────────────────────┘

┌─ 运行时（每次对话）──────────────────────────┐
│                                              │
│  读取当前角色 emotion.json + 文件列表           │
│    → 动态构建情绪指令（仅包含已有文件的情绪）    │
│    → 注入 system prompt                      │
│                                              │
│  User 发送消息                                │
│    → WebSocket → chat_ws.py                  │
│    → agent_loop → LLM                        │
│  ← LLM 流式回复（文本 + [EMOTION:xxx]）       │
│  ← chat_ws.py 解析 → WS 发送 emotion 事件      │
│  ← useChat 更新 lastEmotion                  │
│  ← EmotionPortrait 读取对应情绪图片              │
│     根据 lastEmotion 切换 <img src>               │
│     角色未配置 → 不显示画像                    │
│                                              │
│  ★ 角色切换时                                 │
│    → chat_ws.py 发送当前角色情绪配置             │
│    → EmotionPortrait 重新加载新角色图片          │
│    → lastEmotion 重置为 neutral                │
└──────────────────────────────────────────────┘
```

### 3.2 后端

#### 动态情绪指令

LLM 情绪指令不再硬编码 6 种，而是根据已配置的情绪动态生成：

```
# 角色已配置: neutral, happy, thinking (文件存在)

在每次回复的末尾，用 [EMOTION:xxx] 标记你当前的处理状态。
可选值（仅限以下）：neutral / happy / thinking
选择最接近你当前状态的一个。此标记仅用于 UI 显示，不代表真实情感。
```

```
# 角色未配置任何情绪帧

→ 不注入情绪指令，LLM 不做情绪标记
```

#### chat_ws.py 处理

- **角色切换时**：发送当前角色情绪配置事件 `{"type": "emotion_config", "available": ["neutral", "happy", ...], "enabled": true}`，前端据此加载新角色图片并重置画像
- 检测 LLM 回复末尾的 `[EMOTION:xxx]` 标记
- 校验 key 是否在当前角色的有效情绪列表中
- 从最终回复文本中**移除**该标记（用户不可见）
- 通过 WebSocket 发送事件：`{"type": "emotion", "key": "xxx"}`
- 未检测到标记 → 不发送 emotion 事件（画像保持上一次状态，或显示 neutral）

### 3.3 前端

#### 角色编辑页（RolePage）

| 组件 | 说明 |
|------|------|
| `EmotionToggle` | "情绪画像" 开关，默认关闭 |
| `EmotionSlotGrid` | 6 槽位网格，每格显示预览图或虚线空框 |
| `EmotionCropModal` | 裁剪弹窗：图片导入 + 固定 64×64 选框 + 实时预览 |

#### 聊天页（ChatPage）

| 组件 | 说明 |
|------|------|
| `EmotionPortrait` | 读取当前角色对应情绪图片，根据 emotion key 切换 `<img src>`；角色未配置时 `display: none` |

#### 状态管理

```typescript
type EmotionKey = 'neutral' | 'happy' | 'thinking' | 'confused' | 'working' | 'concerned';

interface EmotionEvent {
  type: 'emotion';
  key: EmotionKey;
}

// ChatState 新增
lastEmotion: EmotionKey | null  // null = 当前角色未配置情绪画像
```

## 五、合规边界

本功能遵循 **"趣味性 UI 反馈"** 定位：

- ✅ 情绪仅通过像素画像的视觉切换表达，**不在文字回复中描述情感**
- ✅ 情绪不跨会话持久化
- ✅ 仅使用中性/功能性情绪，不使用爱、悲伤、孤独等深度情感
- ✅ 画像旁边标注 "AI 状态"，明确非人类身份
- ✅ 用户可随时在设置中关闭（全局 + 角色级双重开关）
- ❌ 不做 AI 伴侣/虚拟恋人等定向功能
- ❌ 不让 AI 说 "我很难过"、"我好想你" 等情感话语
- ❌ 不记忆用户的情感偏好，不做情绪化回复策略
