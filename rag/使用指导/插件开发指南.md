# AICraft 插件开发指南

## 概述

AICraft 的能力扩展有三条路径：**Skill**（提示词模块）、**MCP**（工具调用模块）、**RAG**（知识库模块）。三者互补：

| 类型 | 本质 | 能做什么 | 不能做什么 |
|------|------|---------|-----------|
| Skill | 纯文本 prompt | 改变 AI 的回答风格和专业度 | 执行任何实际操作 |
| MCP | 外部工具服务 | 读写文件、调 API、跑代码 | 影响 AI 的说话风格 |
| RAG | 文档知识库 | 提供领域知识辅助回答 | 执行操作或改变风格 |

---

## 一、开发 Skill

### 什么是 Skill

Skill 是一个包含 `SKILL.md` 文件的文件夹。AICraft 扫描 Skill 目录，将已启用 Skill 的描述注入到 system prompt 中，引导 AI 按特定方式回答。

**Skill 不执行代码，只是告诉 AI "你具备这个能力/按这个风格回答"。**

### 目录结构

```
skills/
├── toggles.json          # 开关状态（自动生成）
├── 翻译助手/
│   └── SKILL.md          # 必须存在
├── 代码审查/
│   └── SKILL.md
└── 自定义技能/
    └── SKILL.md
```

### SKILL.md 格式

```markdown
# 技能名称

一句话描述这个技能做什么（这行会被提取为 Skill 描述，显示在界面上）。

## 规则
- 具体规则1
- 具体规则2

## 输出格式
- 格式要求1
- 格式要求2
```

**要点**：
- 第一段非标题行会被提取为 Skill 的简短描述（显示在管理页面）
- 内容越具体越好，模糊的指令 AI 执行效果差
- 不要写"你是一个XX助手"这种废话，直接写规则和输出要求

### 示例：代码审查 Skill

```markdown
# 代码审查

你是一位严格的代码审查员，从安全、性能、可维护三个维度审查代码。

## 审查维度

### 安全（Security）
- SQL注入、XSS、命令注入等注入风险
- 敏感信息硬编码（密钥、密码、Token）
- 不安全的反序列化、文件操作

### 性能（Performance）
- O(n²)及以上复杂度可优化的地方
- 不必要的重复计算、重复查询
- 内存泄漏风险

### 可维护（Maintainability）
- 函数/方法过长（>50行）
- 嵌套过深（>3层）
- 魔法数字/字符串未提取常量

## 输出格式
按严重程度分级：
- 🔴 严重：必须修复
- 🟡 建议：推荐修复
- 🔵 参考：可选优化

每条包含：位置 → 问题描述 → 修复建议
```

### Skill 注入机制

AICraft 的 `skill_loader.py` 工作流程：
1. 扫描 Skill 目录下所有含 `SKILL.md` 的子目录
2. 提取每个 Skill 的名称和描述
3. `build_skill_prompt()` 将已启用 Skill 的名称+描述拼接到 system prompt
4. AI 根据这些描述调整回答方式

**注意**：只有名称和简短描述被注入，完整的 SKILL.md 内容不会注入到 prompt。所以描述要精炼，规则要可由名称和描述传达。

### 开发建议

- **一个 Skill 做一件事**，不要把翻译和写作塞同一个 Skill
- **描述要精准**，"翻译助手"比"语言专家"好
- **规则要具体**，"输出中英对照格式"比"翻译要专业"好
- **命名用中文**，用户看到的是文件夹名

## 二、接入 MCP

### 什么是 MCP

MCP（Model Context Protocol）让 AI 能调用外部工具。AICraft 通过 stdio 或 SSE 协议连接 MCP 服务器，将工具列表暴露给 AI 的 function calling。

### 两种连接模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **stdio** | 启动子进程，通过标准输入输出通信 | 本地命令行工具 |
| **SSE** | 连接 HTTP 服务器，通过 Server-Sent Events 通信 | 远程服务 |

### 添加 MCP 连接

1. 点击「MCP」标签
2. 点击「添加 MCP」
3. 填写：
   - **名称**：连接显示名
   - **类型**：stdio 或 sse
   - **stdio 模式**：填写 command 和 args（如 `npx @modelcontextprotocol/server-filesystem workspace`，路径相对于项目根目录）
   - **sse 模式**：填写 URL（如 `http://localhost:3001/sse`）
4. 点击添加，然后连接

### 使用官方 MCP 服务器

官方 MCP 服务器（需要 Node.js 环境）：

```bash
# 文件系统读写
npx @modelcontextprotocol/server-filesystem /path/to/workspace

# Python 代码执行
npx @modelcontextprotocol/server-python

# GitHub 操作
npx @modelcontextprotocol/server-github
```

添加时配置：
- 类型：stdio
- command：npx
- args：对应包名和参数

### 自建 MCP Server（stdio）

用 Python 快速创建一个 MCP 服务器：

```python
# my_tool_mcp.py
import json
import sys

def handle_request(request):
    """处理 MCP 请求"""
    method = request.get("method", "")
    
    if method == "initialize":
        return {
            "capabilities": {"tools": {}},
            "serverInfo": {"name": "my-tool", "version": "1.0.0"}
        }
    
    if method == "tools/list":
        return {
            "tools": [
                {
                    "name": "my_function",
                    "description": "我的自定义工具",
                    "inputSchema": {
                        "type": "object",
                        "properties": {
                            "input": {"type": "string", "description": "输入参数"}
                        },
                        "required": ["input"]
                    }
                }
            ]
        }
    
    if method == "tools/call":
        tool_name = request.get("params", {}).get("name", "")
        arguments = request.get("params", {}).get("arguments", {})
        
        if tool_name == "my_function":
            result = do_something(arguments["input"])
            return {"content": [{"type": "text", "text": result}]}
    
    return {}

def do_something(input_text):
    """你的工具逻辑"""
    return f"处理结果: {input_text}"

# MCP stdio 协议：读 JSON-RPC，写 JSON-RPC
for line in sys.stdin:
    line = line.strip()
    if not line:
        continue
    try:
        request = json.loads(line)
        response = {
            "jsonrpc": "2.0",
            "id": request.get("id"),
            "result": handle_request(request)
        }
        print(json.dumps(response), flush=True)
    except Exception as e:
        error_response = {
            "jsonrpc": "2.0",
            "id": request.get("id"),
            "error": {"code": -32603, "message": str(e)}
        }
        print(json.dumps(error_response), flush=True)
```

添加到 AICraft：
- 类型：stdio
- command：python
- args：my_tool_mcp.py

### MCP 工具调用流程

1. AICraft 连接 MCP 服务器，获取工具列表
2. 工具列表注入 AI 的 function calling
3. AI 判断需要调用工具时，生成工具调用请求
4. AICraft 转发请求到 MCP 服务器
5. MCP 服务器执行并返回结果
6. AI 根据结果继续回答

### 安全建议

- 限制文件访问范围，不要暴露整个磁盘
- 代码执行 MCP 应使用沙箱环境
- 不要在 MCP 环境变量中硬编码密钥
- 使用开关管理，不需要时及时关闭

## 三、配置 RAG 知识库

### 什么是 RAG

RAG（检索增强生成）让 AI 基于你的私有文档回答问题，而不是仅靠训练数据。

### 添加知识库

1. 准备文档目录，放入支持的文件（txt/md/py/json/csv/html/xml/docx/pdf）
2. 在 RAG 页面添加数据源，指定目录路径
3. 点击「索引」建立向量索引
4. 对话时开启 RAG检索 开关

### 索引机制

- 使用 ChromaDB 做向量存储
- 文档按 500 字符分块，50 字符重叠
- 首次索引时下载 Embedding 模型，可能需要等待
- 每次点击「索引」会全量重建索引

### 知识库最佳实践

- **文档要结构化**：用标题、列表、表格组织内容
- **一份文档一个主题**：不要把无关内容混在一起
- **关键信息前置**：重要结论放在文档开头
- **定期重建索引**：文档更新后需要重新索引
- **文件名有意义**：文件名会被纳入检索范围

### 适合做 RAG 的内容

- 项目文档和 API 手册
- 公司内部知识库
- 技术规范和标准
- 产品使用说明
- FAQ 常见问题集

### 不适合做 RAG 的内容

- 超大文件（>10MB）：索引慢，检索噪声多
- 图片/视频：不支持
- 频繁变更的临时数据：需要频繁重建索引
