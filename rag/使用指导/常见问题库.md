# AICraft 常见问题库

## 安装与启动

### Q: AICraft 支持哪些操作系统？
A: 目前仅支持 **Windows 10 及以上**。AICraft 使用 Windows 原生窗口框架（pywebview + Win32 API），暂不支持 macOS 和 Linux。

### Q: 启动时报错"未配置模型"
A: 第一次使用需要先在「模型」页添加 API 配置。至少添加一个模型并设为默认。

### Q: 启动后页面空白
A: 检查后端是否正常启动。查看终端输出是否有报错。常见原因：
- 端口 8765 被占用：关闭占用进程或等待应用自动重试（最多 10 次）
- Python 依赖缺失：运行 `pip install -r requirements.txt`

### Q: MCP 连接失败
A: 检查以下几点：
- **Stdio 模式**：确认 command 可执行（如 `npx` 是否在 PATH 中）
- **SSE 模式**：确认目标服务正在运行，URL 可访问
- 查看连接卡片上的错误信息，根据提示排查

### Q: MCP 需要 Node.js 环境
A: 出厂预置的「文件管理」和「代码执行」MCP 通过 `npx` 启动，需要 Node.js。安装步骤：
1. 访问 https://nodejs.org/ 下载 LTS 版本
2. 安装后重启 AICraft
3. MCP 页面状态应变为"环境就绪"

> 自定义 MCP（如 Python 脚本）不需要 Node.js。

### Q: 如何备份数据或迁移到另一台电脑？
A: AICraft 采用便携式设计，所有用户数据存储在 exe/项目根目录下：
- `config/`、`models/`、`memory/`、`chroma_db/`、`workspace/`、`roles/`、`rag/` 等目录

备份这些目录即可。迁移到另一台电脑时要注意：
- 路径可能不同，RAG 数据源和 MCP 配置中的路径会自动尝试自愈
- API Key 会随配置文件一起迁移
- 旧版（v1.0.x）数据在 `%APPDATA%/AICraft/`，升级到新版后会自动迁移

## 模型配置

### Q: 支持哪些模型？
A: 支持所有 OpenAI 兼容接口的模型，以及 Anthropic 原生协议的模型：

| Provider | 协议 | 说明 |
|----------|------|------|
| DeepSeek | Anthropic SDK | 推荐，性价比高，支持深度思考和 server-side 搜索 |
| Anthropic Claude | Anthropic SDK | Claude 系列专用 |
| OpenAI | HTTP (OpenAI 兼容) | GPT 系列 |
| OpenRouter | HTTP | 聚合多种模型 |
| 其他 | HTTP | 任何 OpenAI 兼容 API |

### Q: DeepSeek 一键接入怎么用？
A: 在模型页点击「DeepSeek 一键接入」，输入 API Key，系统自动创建 Pro（`deepseek-chat`）和 Flash（`deepseek-chat`）两个模型配置。两个模型共享同一个 Key，通过 tier 标签区分。

### Q: Auto 模式怎么选择模型？（路由规则）
A: Auto 模式基于规则路由，按以下优先级判断（命中即停止）：

| 优先级 | 条件 | 结果 |
|--------|------|------|
| 1 | 开启深度思考 | → Pro |
| 2 | 消息涉及工具操作意图（如"帮我写个脚本""读取那个文件"） | → Pro |
| 3 | 开启 RAG 检索 | → Pro |
| 4 | 包含复杂关键词（分析/设计/代码/优化/架构等） | → Pro |
| 5 | 消息超过 300 字 | → Pro |
| 6 | 包含代码块（``` 或缩进代码） | → Pro |
| 7 | 包含多个问题（多个问号） | → Pro |
| 8 | 以上全部未命中 | → Flash |

> 核心原则：Pro 兜底，Flash 只在明确简单时用。宁可多花一点，不要答错。

### Q: Tier（Pro/Flash）是什么？怎么设置？
A: Tier 是模型的分级标签，用于 Auto 路由选择。在模型配置中设置：
- **Pro**：处理复杂任务的主力模型
- **Flash**：处理简单问答的轻量模型（通常更快更便宜）

如果你只有一个模型，不需要设置 tier——Auto 路由会直接使用它。

### Q: API Key 存储安全吗？
A: API Key 以明文形式存储在本地 JSON 配置文件中（`models/` 目录下）。AICraft 不会将 Key 上传到任何第三方服务器。建议：
- 不要在公共电脑上保存 Key
- 使用各平台的 API Key 管理功能设置使用限额
- 定期轮换 Key

## 对话功能

### Q: 聊天历史会丢失吗？
A: 不会。聊天历史自动保存到 `memory/conversations/` 目录，重启后可在「记忆」页查看。注意区分：
- **聊天历史**：永久保存到磁盘，重启恢复
- **上下文**：当前对话窗口内有效，新场景时重置
- **记忆**：压缩提取后的长期记忆，跨场景可用

### Q: 切换角色后之前的对话还在吗？
A: 界面上还在（用分割线分隔），但 AI 的上下文会重置。新角色不会继承旧角色的对话语境。如需保留对话内容，建议先用「新场景」归档再切换角色。

### Q: "新场景"是什么？
A: 归档当前对话到记忆，清空上下文，但历史消息保留在界面上用分割线分隔。相当于开启一段全新对话，同时保留之前的记录。

### Q: 深度思考什么时候该开？
A: 适合开启的场景：
- 复杂逻辑推理
- 多步骤决策
- 需要分析多个方案的对比
- 数学/编程问题

不需要开启的场景：
- 简单问答
- 日常闲聊
- 翻译

> 深度思考仅 DeepSeek/Claude 模型支持。开启后推理过程会额外消耗 Token。

### Q: 联网搜索是怎么工作的？需要配置吗？
A: 联网搜索通过模型平台的**原生 server-side 搜索**实现（DeepSeek 和 Claude 原生支持），AI 在需要时会自动调用搜索工具。**无需任何配置**，只要你的模型支持即可。

另外，AICraft 内置了 4 个快捷数据源（天气、金价、汇率、热搜），通过免费公开 API 直接查询，同样无需配置。

### Q: 为什么联网搜索有时候不触发？
A: 是否调用搜索由 AI 模型自行判断。如果你觉得应该搜索但没有触发，可以：
- 在消息中明确要求"请搜索一下..."
- 使用 DeepSeek 模型（server-side 搜索支持最好）

### Q: AI 回答出现幻觉（编造事实）怎么办？
A: AICraft 已内置反幻觉约束，但仍可能发生。建议：
- 开启深度思考，让 AI 先推理再回答
- 开启 RAG，让 AI 基于事实文档回答
- 对时效性信息，明确要求搜索（"请搜索后回答"）
- 对关键数据，要求 AI 标注来源

## Skill 相关

### Q: Skill 和 MCP 有什么区别？
A:
- **Skill** = 提示词模块，改变 AI 的回答风格，**不执行任何实际操作**
- **MCP** = 工具模块，让 AI **真正能读写文件、执行代码等**

举例：告诉 AI "你会代码审查"是 Skill；让 AI 真的能读文件审查代码是 MCP。

### Q: 自定义 Skill 怎么写？
A: 在 Skill 根目录下创建一个文件夹，放入 `SKILL.md` 文件即可。格式详见 [插件开发指南](插件开发指南.md)。

### Q: 为什么 Skill 开了但 AI 没按 Skill 的方式回答？
A: **关键原因：只有 Skill 的名称和第一段描述会被注入到 System Prompt，完整的 SKILL.md 内容不会发给 AI。** 因此：

- 第一段描述必须精准概括 Skill 的能力和回答风格
- 如果一个描述很短但名称为"代码审查"，AI 只知道"你具备代码审查能力"而不知道具体要查什么
- 详细规则可以作为参考文档写在 SKILL.md 中，但要通过名称和描述让 AI 知道"该怎么做"

其他可能原因：
- 多个 Skill 之间有冲突
- 对话上下文太长，Skill 提示被预算裁剪
- 尝试在消息中明确提及"用XX技能回答"

## MCP 相关

### Q: MCP 连接成功但 AI 不调用工具？
A: 检查：
- MCP 是否已启用（开关打开）
- MCP 是否已连接（状态为"已连接"，工具列表有内容）
- 消息是否真的需要工具（AI 自行判断是否调用）
- 在消息中明确要求（如"请用文件管理工具读取 xxx"）

### Q: MCP 工具调用失败？
A: 常见原因：
- 文件路径不存在或无权限
- 代码执行超时
- 工具参数格式不对
- 被权限策略拒绝（查看弹窗提示）
- 查看错误信息，根据提示调整

### Q: MCP 工具执行时弹窗让我确认，怎么控制？
A: 有三种方式管理权限确认：

1. **自动授权**：在 MCP 连接卡片上开启「自动授权」开关，该连接的所有工具都不再弹窗确认（⚠️ 仅限你完全信任的连接）
2. **信任路径**：在 MCP 页面底部的「文件访问权限」面板添加信任路径，匹配的文件操作自动放行
3. **单次允许/拒绝**：弹窗中选择「允许一次」「始终允许」「始终拒绝」等选项

> 代码执行（Python/Shell）始终需要确认，无法完全跳过——这是安全设计。

### Q: 可以同时使用多个 MCP 吗？
A: 可以。所有已启用且已连接的 MCP 的工具会合并提供给 AI，AI 自行判断调用哪个。

### Q: MCP 的「自动授权」和权限策略有什么区别？
A:
- **自动授权**：针对某个 MCP 连接的全局开关，开启后彻底跳过所有权限检查
- **权限策略**：针对文件路径的精细控制（信任/拒绝），适用于特定的目录或文件模式

两者可以组合使用：对不信任的 MCP 连接关闭自动授权，同时配置信任路径让常规操作不弹窗。

## RAG 相关

### Q: 索引很慢怎么办？
A: 取决于 Embedding 模式和文档数量：

| 模式 | 首次索引 | 后续索引 |
|------|---------|---------|
| **local**（本地 ONNX） | 需下载模型（~80MB），之后较快 | 取决于文件数量 |
| **api**（硅基流动） | 立即开始，速度取决于 API 响应 | 同上 |
| **auto** | 与 local 相同（无 Key 时走本地） | 同上 |

- 10 个文件以内：几秒钟
- 100 个文件：可能需要几分钟
- 超大文件或扫描版 PDF 建议预处理

### Q: RAG 检索结果不准确？
A: 改进方法：
- 确保文档内容结构清晰，有明确的标题和段落
- 文档按主题拆分，不要一份文档混合多个主题
- 关键信息放在文档开头
- 尝试切换到 API Embedding 模式（1024 维，比本地 384 维质量更高）
- 文档更新后点击「重新索引」

### Q: Embedding 模式选 auto / api / local 哪个好？
A: 
- **auto**（推荐）：有 API Key 自动用 API，没有则用本地 ONNX。适合大多数用户
- **api**：使用硅基流动的云端 Embedding 模型（1024 维），质量更高，但需要联网和 API Key。免费注册：https://cloud.siliconflow.cn
- **local**：完全离线运行（384 维），不需要网络和 Key。首次使用下载 ~80MB 模型

> ⚠️ 切换模式后必须**重新索引**所有数据源，因为不同模式的向量维度不兼容。

### Q: 支持 PDF 吗？
A: 支持。需要安装 `PyPDF2`：`pip install PyPDF2`。注意：扫描版 PDF（纯图片）无法提取文本，需要 OCR 预处理。

## 记忆相关

### Q: 记忆是怎么工作的？
A: 三层架构，全部自动运行：

1. **L0 实时记忆**：对话中产生的关键信息自动记录
2. **L1 短期压缩**：对话达到一定长度，自动用 Flash 模型压缩为摘要（默认每 20 条消息触发）
3. **L2 长期合并**：压缩片段积累到 8 个，自动合并为长期记忆

### Q: 可以手动编辑记忆吗？
A: 可以。记忆文件保存在 `memory/project-notes/` 目录，为 `.md` 格式，可以直接用文本编辑器修改。长期记忆文件为 `memory/long_term_memory.md`。

### Q: 记忆会占用很多 token 吗？
A: 记忆注入有预算控制（默认最多注入 4000 字符），不会无限膨胀。超出预算时按优先级裁剪（长期记忆 > 短期压缩，最近优先）。

### Q: 可以手动触发记忆合并吗？
A: 可以。在「记忆」页 → 「记忆设置」tab → 状态面板中点击「合并现在」按钮，立即将所有短期压缩片段合并为长期记忆。

## Token 与费用

### Q: 怎么查看 Token 用量？
A: 点击对话页输入区右侧的 📊 图标，展开 Token 用量面板。可以看到：
- 本次会话的输入/输出 Token
- 按模型统计的消耗分布
- 缓存命中情况（DeepSeek 支持 context caching）

### Q: 如何节省 API 费用？
A:
- 使用 **Auto 路由**，让简单问题自动走 Flash 模型（更快更便宜）
- 简单闲聊不要开启**深度思考**（推理过程额外消耗 Token）
- 利用 **记忆压缩**：后台摘要任务自动使用 Flash 模型，不浪费 Pro Token
- 控制**上下文预算**：适当降低 `max_history_chars` 和 `memory_inject_max_chars`
- 不需要 RAG 检索时关闭 RAG 开关

## 性能与故障

### Q: 回复速度慢？
A: 可能原因：
- 使用了 Pro 模型，试试 Auto 路由让简单问题用 Flash
- 开启了深度思考，推理过程需要时间
- RAG 检索耗时（特别是首次查询冷启动 ChromaDB）
- 网络延迟（API 服务器响应慢，特别是海外服务器）
- Embedding 模型首次加载（本地模式）

### Q: 对话中断/报错？
A: 常见原因：
- API Key 余额不足
- 网络不稳定
- 模型上下文窗口超限（查看上下文百分比）
- 后端服务异常（查看终端日志）
- 被权限策略拒绝操作

### Q: 如何查看日志？
A: 后端日志直接输出到启动终端。应用日志写入 `aicraft.log` 文件。

### Q: 如何重置所有配置？
A: 删除 `config/` 目录下的配置文件，重启应用会恢复默认设置。聊天历史和记忆保存在 `memory/` 目录，不会丢失。

> ⚠️ 删除前建议先备份，特别是模型配置中的 API Key。

### Q: 如何重置权限策略？
A: 删除 `config/permissions.json` 文件，重启应用会自动生成默认权限配置。
