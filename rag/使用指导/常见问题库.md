# AICraft 常见问题库

## 安装与启动

### Q: 启动时报错"未配置模型"
A: 第一次使用需要先在「模型」页面添加 API 配置。至少添加一个模型并设为默认。

### Q: 启动后页面空白
A: 检查后端是否正常启动。查看终端输出是否有报错。常见原因：
- 端口 8765 被占用：关闭占用进程或修改 `run.py` 中的端口
- Python 依赖缺失：运行 `pip install -r requirements.txt`

### Q: MCP 连接失败
A: 检查以下几点：
- stdio 模式：确认 command 可执行（如 `npx` 是否在 PATH 中）
- SSE 模式：确认目标服务正在运行，URL 可访问
- 查看连接的错误信息，根据提示排查

### Q: MCP 需要 Node.js 环境
A: 出厂预置的「文件管理」和「代码执行」MCP 需要 Node.js。安装步骤：
1. 访问 https://nodejs.org/ 下载 LTS 版本
2. 安装后重启 AICraft
3. MCP 页面状态应变为"环境就绪"

## 模型配置

### Q: 支持哪些模型
A: 支持所有 OpenAI 兼容接口的模型，包括：
- DeepSeek（推荐，性价比最高）
- OpenAI GPT 系列
- Anthropic Claude 系列（需选 anthropic 协议）
- 通过 OpenRouter 接入的各种模型

### Q: DeepSeek 一键接入怎么用
A: 在模型页点击「DeepSeek 一键接入」，输入 API Key，系统自动创建 Pro（deepseek-chat）和 Flash（deepseek-chat）两个模型配置。Key 在同一提供商下的模型之间共享。

### Q: Auto 模式怎么选择模型
A: Auto 模式基于规则路由，按以下优先级判断：
- 开启深度思考 → Pro
- 有 MCP 工具调用 → Pro
- 开启 RAG → Pro
- 消息包含复杂关键词（分析/设计/优化/架构等）→ Pro
- 消息超过 300 字 → Pro
- 包含代码块 → Pro
- 包含多个问题 → Pro
- 其他情况 → Flash

### Q: 如何切换协议
A: 在模型配置中选择：
- **litellm**：通用协议，支持 OpenAI/DeepSeek/OpenRouter 等
- **anthropic**：Claude 系列专用，使用 Anthropic SDK 原生调用

## 对话功能

### Q: 聊天历史会丢失吗
A: 不会。聊天历史自动保存到磁盘，重启后自动恢复。但注意区分：
- **聊天历史**：永久保存，重启恢复
- **上下文**：当前对话窗口内有效，新场景时重置
- **记忆**：长期保存，跨场景可用

### Q: 切换角色后之前的对话还在吗
A: 在界面上还在（用分割线分隔），但 AI 的上下文会重置。切换角色时系统会自动提取纯事实信息传递给新角色，不会泄露旧角色的说话风格。

### Q: "新场景"是什么
A: 归档当前对话到记忆，清空上下文，但历史消息保留在界面上用分割线分隔。相当于开启一段全新对话，同时保留之前的记录。

### Q: 深度思考什么时候该开
A: 适合开启的场景：
- 复杂逻辑推理
- 多步骤决策
- 需要分析多个方案的对比
- 数学/编程问题

不需要开启的场景：
- 简单问答
- 日常闲聊
- 翻译

### Q: AI 回答出现幻觉（编造事实）怎么办
A: AICraft 已内置反幻觉约束，但仍可能发生。建议：
- 开启深度思考，让 AI 先推理再回答
- 开启 RAG，让 AI 基于事实文档回答
- 对时效性信息，依赖联网搜索而非 AI 记忆
- 对关键数据，要求 AI 标注来源

## Skill 相关

### Q: Skill 和 MCP 有什么区别
A:
- **Skill** = 提示词模块，改变 AI 的回答风格，不执行任何操作
- **MCP** = 工具模块，让 AI 真正能读写文件、执行代码等

举例：告诉 AI "你会代码审查"是 Skill；让 AI 真的能读文件审查代码是 MCP。

### Q: 自定义 Skill 怎么写
A: 在 Skill 根目录下创建一个文件夹，放入 `SKILL.md` 文件即可。文件格式：
```markdown
# 技能名称
一句话描述

## 规则
- 具体规则

## 输出格式
- 格式要求
```

### Q: 为什么 Skill 开了但 AI 没按 Skill 的方式回答
A: 可能的原因：
- Skill 描述不够具体，AI 没有理解
- 多个 Skill 之间有冲突
- 对话上下文太长，Skill 提示被淹没
- 尝试在消息中明确提及"用XX技能回答"

## MCP 相关

### Q: MCP 连接成功但 AI 不调用工具
A: 检查：
- MCP 是否已启用（开关打开）
- MCP 是否已连接（状态为"已连接"）
- 消息是否真的需要工具（AI 自行判断是否调用）
- 在消息中明确要求（如"请用文件管理工具读取xxx"）

### Q: MCP 工具调用失败
A: 常见原因：
- 文件路径不存在或无权限
- 代码执行超时
- 工具参数格式不对
- 查看错误信息，根据提示调整

### Q: 可以同时使用多个 MCP 吗
A: 可以。所有已启用的 MCP 的工具会合并提供给 AI，AI 自行判断调用哪个。

## RAG 相关

### Q: 索引很慢怎么办
A: 首次索引需要下载 Embedding 模型，可能较慢。后续索引取决于文档数量：
- 10 个文件以内：几秒钟
- 100 个文件：可能需要几分钟
- 超大文件建议拆分

### Q: RAG 检索结果不准确
A: 改进方法：
- 确保文档内容结构清晰，有明确的标题和段落
- 文档按主题拆分，不要一份文档混合多个主题
- 关键信息放在文档开头
- 文档更新后重新索引

### Q: 支持 PDF 吗
A: 支持。安装 PyPDF2 即可：`pip install PyPDF2`。注意扫描版 PDF（图片形式）无法提取文本。

## 记忆相关

### Q: 记忆是怎么工作的
A: 三层架构：
1. **实时记忆**：对话中产生的关键信息
2. **短期压缩**：对话达到一定长度，自动压缩为摘要（使用 Flash 模型节省成本）
3. **长期合并**：压缩片段积累到一定数量，自动合并为长期记忆

### Q: 可以手动编辑记忆吗
A: 可以。在「记忆」页面查看所有记忆笔记，可以手动编辑内容。

### Q: 记忆会占用很多 token 吗
A: 记忆注入有预算控制（默认 4000 字符），不会无限膨胀。超出预算时按优先级裁剪。

## 性能与故障

### Q: 回复速度慢
A: 可能原因：
- 使用了 Pro 模型，试试 Auto 路由让简单问题用 Flash
- 开启了深度思考，推理过程需要时间
- RAG 检索耗时
- 网络延迟（API 服务器响应慢）

### Q: 对话中断/报错
A: 常见原因：
- API Key 余额不足
- 网络不稳定
- 模型上下文窗口超限（查看上下文百分比）
- 后端服务异常（查看终端日志）

### Q: 如何查看日志
A: 后端日志直接输出到启动终端。前端问题可按 F12 打开开发者工具查看控制台。

### Q: 如何重置所有配置
A: 删除 `config/` 目录下的配置文件，重启应用会恢复默认设置。聊天历史和记忆保存在 `memory/` 目录，不会丢失。
