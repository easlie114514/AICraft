# AICraft 使用手册

## 什么是 AICraft

AICraft 是一个桌面 AI 能力启动器。像游戏启动器管理 mod 一样，AICraft 用可视化界面管理大语言模型的 Skill、MCP、RAG、记忆等能力模块，让 AI 从聊天工具变成真正的生产力搭档。

## 快速上手

### 1. 启动应用

双击 `AICraft.exe` 或运行 `python run.py`，应用会自动启动后端服务并打开桌面窗口。

### 2. 配置模型

第一次使用需要配置模型 API：

1. 点击顶部「模型」标签
2. 点击「添加模型」
3. 填写以下信息：
   - **名称**：给模型取个名字，如"DeepSeek Pro"
   - **模型ID**：模型标识符，如 `deepseek/deepseek-chat`，格式为 `提供商/模型名`
   - **提供商**：deepseek / openai / anthropic / openrouter 等
   - **API Key**：对应平台的密钥
   - **API Base**（可选）：自定义接口地址
   - **协议**：litellm（通用）/ anthropic（Claude 系列专用）
4. 设为默认模型

**DeepSeek 一键接入**：在模型页点击「DeepSeek 一键接入」，填入 API Key 即可自动创建 Pro + Flash 两个模型配置。

### 3. 选择角色

点击对话页输入框右侧的角色下拉框，选择预设角色。角色决定了 AI 的说话风格和专业领域。

### 4. 开始对话

在输入框输入消息，按 Enter 发送（Shift+Enter 换行）。对话页输入框右侧有三个开关：

| 开关 | 作用 |
|------|------|
| **RAG检索** | 开启后 AI 会从知识库检索相关信息辅助回答 |
| **记忆注入** | 开启后 AI 会参考历史对话记忆（默认开启） |
| **深度思考** | 开启后 AI 会先进行深度推理再回答 |

## 核心功能

### 多模型自由切换

在输入框右侧的模型下拉框切换模型，支持：

- **手动切换**：直接选择某个模型
- **Auto（智能路由）**：AI 根据问题复杂度自动选择 Pro 或 Flash 模型。复杂问题（代码、多步骤推理、工具调用）自动升级 Pro，简单闲聊降级 Flash 省钱

### 多角色快切

切换角色时，AICraft 会自动：
- 提取当前对话的纯事实信息（去掉旧角色的说话风格）
- 清除旧角色上下文
- 注入新角色的设定和风格

### 深度思考

开启深度思考后，AI 会在回答前进行推理分析，推理过程以折叠形式展示。适用于：
- 复杂问题拆解
- 逻辑推理
- 多步决策

### 联网搜索

AI 会根据需要自动调用联网搜索，无需手动触发。搜索结果会标注来源链接。

**快捷数据源**：搜索时会优先查询权威来源：
- 天气 → 中国天气网
- 金价/汇率 → 东方财富
- 股票/基金 → 东方财富/同花顺
- 新闻 → 新华社/央视新闻

### Skill 管理

Skill 是纯文本提示词模块，改变 AI 的回答方式和风格。

1. 点击顶部「Skill」标签
2. 设置 Skill 根目录路径
3. 每个子目录（含 SKILL.md）即为一个 Skill
4. 用开关启用/禁用 Skill

出厂预置 4 个 Skill：翻译助手、代码审查、写作助手、数据分析。

### MCP 管理

MCP（Model Context Protocol）是工具调用模块，让 AI 真正能执行操作。

1. 点击顶部「MCP」标签
2. 添加 MCP 连接（支持 SSE 和 Stdio 两种模式）
3. 启用后 AI 可调用对应工具

出厂预置 2 个 MCP：文件管理（读写文件）、代码执行（运行 Python）。

### RAG 知识库

RAG（检索增强生成）让 AI 基于你的私有文档回答问题。

1. 点击顶部「RAG」标签
2. 点击「添加数据源」
3. 填写名称和本地文档目录路径
4. 点击「索引」建立向量索引
5. 对话时开启 RAG检索 开关

支持的文件类型：txt、md、py、json、csv、html、xml、docx、pdf

### 长期记忆

AICraft 的记忆系统自动运行：

- **实时记忆**：对话中产生的关键信息自动记录
- **短期压缩**：对话达到一定长度后，自动压缩为摘要
- **长期合并**：多个压缩片段自动合并为长期记忆
- **记忆注入**：新对话自动参考历史记忆

可在「记忆」页面查看和管理记忆笔记。

### 上下文预算

AICraft 自动管理发送给 AI 的上下文窗口：

- 6 级优先级：角色约束（不可裁剪）> Skill > RAG > 记忆 > 跨会话记忆 > 聊天历史
- 超出模型上下文窗口时，按优先级从低到高裁剪
- 对话页显示当前上下文使用百分比

### 新场景

点击「新场景」按钮可以重置对话上下文：
- 当前场景自动压缩归档到记忆
- 开始全新的对话上下文
- 历史消息保留在界面上（用分割线分隔）

## 配置说明

配置文件位于 `config/profiles/default/` 目录：

| 文件 | 内容 |
|------|------|
| `model.json` | 当前选中的模型、角色、上下文参数 |
| `app.json` | 应用全局设置（主题、语言、Skill目录） |

上下文参数说明：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| max_history_chars | 50000 | 聊天历史最大字符数 |
| memory_compact_enabled | true | 是否启用记忆压缩 |
| memory_compact_trigger | messages | 触发方式：chars/messages/both |
| memory_compact_interval_msgs | 20 | 每N条消息触发压缩 |
| memory_compact_max_tokens | 800 | 压缩摘要最大token数 |
| memory_merge_threshold | 8 | N个压缩片段触发合并 |
| memory_inject_max_chars | 4000 | 记忆注入最大字符数 |
| cross_session_inject_count | 10 | 跨会话注入消息条数 |
| context_budget_enabled | true | 是否启用上下文预算管理 |
| budget_alert_threshold | 0.75 | 预算告警阈值（75%时显示） |

## 常用操作

- **复制消息**：点击消息下方的「复制」按钮
- **停止生成**：生成中点击红色方块按钮
- **查看思考过程**：点击「思考过程」折叠区域展开
- **查看上下文用量**：查看输入区上方的百分比徽章
